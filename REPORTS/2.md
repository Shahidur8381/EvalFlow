# EvalFlow Sprint 2 Report: Role-Based Access Control (RBAC)

## Overview
Successfully implemented the Role-Based Access Control (RBAC) system for EvalFlow. The platform now supports three distinct user roles: **Admin**, **Evaluator**, and **Student**, complete with isolated dashboards and robust routing security.

## Features Completed
- **Database Schema Updates**: Added a \ole\ column to the \users\ table.
- **Dynamic Routing**: The core \/dashboard\ route now automatically identifies the user's role upon login and redirects them to their respective dedicated panel.
- **Role Middleware**: Created a custom \RoleMiddleware\ that intercepts incoming requests and strictly blocks unauthorized access. (e.g., a student cannot access the admin URL).
- **Isolated Views**: Scaffolded three distinct dashboard views for Admin, Evaluator, and Student.
- **Database Seeding**: Created default dummy accounts for seamless local testing.

## Tech Stack Used
- **Backend Framework**: Laravel 11.x
- **Authentication**: Laravel Breeze
- **Database**: MySQL (via XAMPP)
- **Frontend Template**: Blade, Tailwind CSS
