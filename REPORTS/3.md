# EvalFlow Sprint 2 Part 2 Report: Core Data & Evaluation Workflow

## Overview
Successfully implemented the core workflow for EvalFlow, moving from exam creation to script submission and finally evaluation. The three main roles now have their primary functional loops completed.

## Features Completed
- **Admin Panel:** Admins can now create Courses and Exams. Exams have a strict time constraint (start/end time) and a total marks field.
- **Student Panel:** Students can view active exams. If they are within the allowed time window, they can upload their answer scripts (restricted to PDF files only).
- **Evaluator Panel:** Evaluators can view a list of submitted scripts. Clicking on a script opens a split-screen grading view featuring an embedded PDF viewer for the student's submission on the left and a form to assign marks on the right.
- **Storage Configuration:** Configured local public storage (`php artisan storage:link`) to securely host and serve the uploaded PDF files.

## Tech Stack Used
- **Backend Framework:** Laravel 11.x
- **Database:** MySQL
- **Frontend Template:** Blade, Tailwind CSS
- **PDF Viewer:** Native HTML `<iframe/>` integration

## How to Test
1. Log in as **Admin** (`admin@evalflow.com`) and create a Course.
2. Still as Admin, create an Exam. Set the Start Time slightly in the past, and the End Time slightly in the future to simulate an ongoing exam.
3. Log out and log in as **Student** (`student@evalflow.com`). You will see the ongoing exam with an 'Upload PDF' form. Upload a PDF file.
4. Log out and log in as **Evaluator** (`evaluator@evalflow.com`). You will see the submitted script in the 'Pending Evaluation' list.
5. Click 'Grade Script', view the PDF inline, and assign a mark.

## New API / Routes
| Method | URI | Description | Role Required |
|--------|-----|-------------|---------------|
| POST | `/admin/courses` | Store a new course | `admin` |
| POST | `/admin/exams` | Store a new exam | `admin` |
| POST | `/student/scripts/{exam}/upload` | Upload a PDF answer script | `student` |
| GET | `/evaluator/scripts/{script}/grade` | View the PDF and grading interface | `evaluator` |
| POST | `/evaluator/scripts/{script}/grade` | Save the assigned marks | `evaluator` |

## Updates & Bug Fixes
- **Timezone Issue:** The default application timezone has been configured to `Asia/Dhaka` so that exam start and end times correspond to your local timezone accurately.
- **Admin Validation Display:** Added an error message display to the Admin Dashboard. If an admin tries to enter an `end_time` that is before the `start_time`, they will now receive a clear validation error blocking the submission.
- **Admin Exam List:** The Admin Dashboard now correctly lists all created exams (past, present, and future) below the creation forms.
