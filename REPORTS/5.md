# EvalFlow Sprint 4 Report: User Management, Question Media & UX Hardening

---

## Overview
Sprint 4 focuses on expanding admin control, tightening student-facing security rules, and improving the overall user experience across all three roles. Key additions include a full user management panel for the admin, optional PDF media attachments per question, inline PDF preview for students, and further UX polish such as search/filter, CamScanner instructions, and paper-locking before exam time.

## Features Implemented

### 👥 Admin — User Management
- **Create Evaluator Accounts** — Admin can register new evaluator accounts directly from the Admin panel without going through the public registration flow. Password is validated with the same strength rules as normal registration.
- **Delete Any User (Student or Evaluator)** — Admin can delete any student or evaluator account from a single management page. Deleting a student cascades cleanly through their submitted scripts. Deleting an evaluator removes their assignment from active exams without deleting the exams themselves. Admin accounts cannot be deleted for safety.
- **Search & Filter** — The user list is filterable by role (`All Roles`, `Evaluators`, `Students`) and searchable by name or email via a live GET-form filter. A "Clear Filters" button appears when a filter is active.
- **Sidebar Navigation** — A new "👥 Users" link was added to the admin sidebar, with active-route highlighting.

### 📎 Admin — Question PDF Media Attachment
- **Optional Media Upload** — When adding a question to an exam, the admin can optionally attach a PDF file (max 20 MB). The upload field sits side-by-side with the Marks field for a compact, clean layout.
- **Stored Securely** — Files are stored in the `questions_media/` folder on the `public` disk via Laravel's `store()` method.
- **Visible in Admin Panel** — If a question has an attachment, a "📄 View Media" button appears inline under the question body in the questions list table.

### 📚 Student — Exam Page UX
- **Paper Locked Before Exam** — Students visiting the exam detail page before the start time see a lock icon and the message "The question paper will be revealed when the exam starts." The question list is completely hidden server-side; it is not just CSS.
- **"View Details" Instead of "Preview Paper"** — The dashboard link for upcoming exams was renamed to avoid confusion; it no longer implies access to the question content.
- **Question PDF Media** — If a question has an attached PDF, students see an "📄 Open Attached PDF" button below the question text to open it in a new tab.
- **CamScanner Instructions** — The upload instructions now include: *"Write your answers on paper. Click photos with a PDF generator app (e.g. CamScanner) and upload the PDF."* to guide students on the expected submission workflow.
- **Inline PDF Preview of Submitted Script** — After submitting, the student can now see an embedded iframe preview of their uploaded answer script directly on the exam page, alongside an "↗ Open Full" button.

### ✅ Evaluator — Question Media
- **Question PDF Links in Grading View** — On the per-question grading panel, if a question has an attached PDF, evaluators see a "📄 View Question PDF" button inline, styled in the evaluator dark theme.

## Middleware / Validation Changes
| Validation | Location | What It Does |
|-----------|---------|-------------|
| `nullable\|file\|mimes:pdf\|max:20480` | `addQuestion()` | Rejects non-PDF or oversized media attachments. |
| Admin deletion guard | `destroy()` in `AdminUserController` | Blocks deletion of any user with `role === 'admin'`. Returns HTTP 403. |
| Search & filter query builder | `index()` in `AdminUserController` | Applies `like` search on name/email and exact match on role, gracefully returning all users if no filters are set. |

## How to Test (End-to-End)

1. **Admin → Users Page:** Log in as Admin. Click "👥 Users" in the sidebar.
2. **Create Evaluator:** Fill in the form with name, email, and password. Submit. The evaluator appears in the user list.
3. **Filter by Role:** Select "Evaluators" from the dropdown and click Filter. Only evaluator accounts should appear. Try "Students" next.
4. **Search:** Type part of a name or email in the search box and filter. Matching results appear.
5. **Delete a User:** Click "Delete" next to a student or evaluator. Confirm the dialog. The account is removed.
6. **Add Question with PDF:** Open an exam, add a question, and attach a PDF via the Media field. Save. Verify "📄 View Media" appears in the questions list.
7. **Student → Upcoming Exam:** Log in as Student. Click "View Details" on an upcoming exam. The question paper should be hidden behind a lock message.
8. **Student → Active Exam:** Once the exam is active, revisit the page. The full question paper now appears, including "📄 Open Attached PDF" for questions with media.
9. **Student → Submit & Preview:** Upload a PDF script. After submission, the exam page should show your uploaded PDF embedded in an iframe.
10. **Evaluator → Check Media Links:** Log in as the assigned Evaluator. Open the grading form. Verify "📄 View Question PDF" links appear for questions with attachments.
