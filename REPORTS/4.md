# EvalFlow Sprint 3 Report: Full System Overhaul

---

## Overview
Sprint 3 is the most significant sprint yet. It transforms EvalFlow from a basic file-upload system into a fully functional examination evaluation platform with structured question papers, strict server-side business rules, role-gated evaluator access, and a completely redesigned professional UI.

## Features Implemented

### 🏠 Admin Panel
- **Exam Creation** — Create exams with Course, Title, Start Time, and End Time. Time window enforced by middleware before reaching the controller.
- **Exam Deletion** — Delete an exam (cascades to all submitted scripts).
- **Question Management** — Add questions with body text and marks. Remove individual questions. Total marks auto-calculated from sum of all questions' marks.
- **Exam List** — Full list of all exams (upcoming, active, past) ordered by newest first with status badges.
- **Evaluator Assignment** — Assign any registered evaluator to an exam from the Exam Detail page. Only the assigned evaluator can see and grade that exam's scripts.

### 📚 Student Panel
- **Exam Overview** — Categorized into Active, Upcoming, and Past sections.
- **Question Paper** — Full question paper shown when viewing an active or upcoming exam.
- **PDF Upload** — Enforces one submission per exam; blocked outside the time window.
- **Submission Tracking** — Real-time status: Upcoming → Active/Upload → Submitted (Pending) → Evaluated (Mark: xx/xx).

### ✅ Evaluator Panel
- **Role-Gated Access** — Evaluator can ONLY see scripts from exams they are personally assigned to. Unassigned evaluators see an empty dashboard with a helpful message.
- **Progress Stats** — Dashboard shows total scripts, pending, evaluated, and a percentage progress.
- **Grading View** — Split-screen: full-height PDF viewer on the left; question reference, grading form, and student info panel on the right.

### 🎨 UI / Design System
- Full professional dark sidebar replacing the Tailwind navbar.
- Custom CSS design system with tokens (colors, spacing, badges, tables, alerts).
- Glassmorphism login/register pages with dark gradient background.
- Status badges (green/yellow/gray/blue/purple/red).
- Stat cards on all dashboards.

## How to Test (End-to-End)

1. **Admin → Create Course:** Log in as Admin, go to Dashboard, fill in Course Name + Code.
2. **Admin → Create Exam:** Fill in Exam details. Try setting End Time before Start Time — you'll see a validation error from the middleware.
3. **Admin → Add Questions:** Click "Manage" on the exam. Add 3–4 questions with different marks. Observe the Total Marks auto-updating.
4. **Admin → Assign Evaluator:** On the exam detail page, select an evaluator from the dropdown and save.
5. **Student → View Exam:** Log in as Student. The active exam shows with a "View Paper & Upload" button. Click it to see the full question paper.
6. **Student → Upload Script:** Select a PDF file and submit. You'll be redirected to the dashboard with a "Submitted" status.
7. **Evaluator A → Check Dashboard:** Log in as Evaluator A (the one assigned). You see the student's script. Click "Grade", view the PDF, assign marks, save.
8. **Evaluator B → Check Dashboard:** Log in as a *different* evaluator. Dashboard shows "No scripts assigned to you" — role gating works.
9. **Student → See Result:** Log back in as Student. The exam now shows "Evaluated · Mark: xx/xx".
